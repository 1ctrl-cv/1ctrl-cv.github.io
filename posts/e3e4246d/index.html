A卡在Arch下安装FAH7.6.21

发表于2026-03-02|更新于2026-03-02
|字数总计:829|阅读时长:3分钟|阅读量:|评论数:

前言

用 AUR 助手安装会导致某些问题，所以我写了这篇文章

环境：

  • CPU：AMD A8-7410
  • GPU：AMD Radeon R5 M330
  • OS：CatOS rolling x64

非常老的设备了，毕竟只是演示用的

驱动参考这篇 archlinux 显卡驱动，写的很详细

安装

在AUR的存档中下载 旧版 的仓库并解压，注意直接 makepkg -si 会报错，因为 FAH 官网删除了旧版所需的文件

在互联网档案馆中可以找到所需的 文件，直接放在刚才解压的仓库文件夹里，然后重新构建软件包

记得装完以后执行 systemctl daemon-reload

配置

运行 FAHClient --configure，以 root 身份生成 /etc/foldingathome/config.xml 配置文件。你也可以手动编辑它。

然后启用并运行 foldingathome.service 单元。

运行

图形化运行

AUR助手可以安装FAHControl，或者打开浏览器访问 http://localhost:7396/

终端中运行

foldingathome 的行为可以通过编辑 /opt/fah/config.xml 来定制。可以指定的一些选项如下：

  • 密码，用于识别你的身份。虽然没什么用
  • CPU 或 GPU 并行

提示GPU被禁用（来自ArchWiki，机器翻译）

Navi 10 based GPUs and later suffer from ROCm conflicting with Folding@home’s shipped libstdc++ library. In order to fix this, try to apply this temporary workaround:
基于 Navi 10 及更新版本的显卡存在 ROCm 与 Folding@home 预装 libstdc++ 库冲突的问题。为解决此问题，请尝试应用此临时修复方案：

  • start Folding@home, and wait for the GPU to fail;
  • 启动 Folding@home，并等待显卡出现故障；

the GPU slot may need to be re-enabled for this.
这个 GPU 插槽可能需要重新启用。

  • replace Folding@home’s current core’s libstdc++.so.6 by your system’s own library, e.g., /usr/lib/libstdc++.so.6.
  • 将 Folding@home 当前核心的 libstdc++.so.6 替换为系统自带的库，例如 /usr/lib/libstdc++.so.6。

Folding@home’s core can be found at /var/lib/fah/cores/cores.foldingathome.org/lin/64bit/.
Folding@home 的核心位于 /var/lib/fah/cores/cores.foldingathome.org/lin/64bit/。

Note: you need superuser privileges to enter these directories.
注意：进入这些目录需要超级用户权限。

The fix will stop working when a new Folding@home core version is released: it will need to be reapplied.
修复将在发布新的 Folding@home 核心版本时失效：需要重新应用。

If it’s not working yet, try editing /etc/foldingathome/config.xml file to attempt to autoconfigure GPUs:
如果尚未生效，尝试编辑 /etc/foldingathome/config.xml 文件以尝试自动配置 GPU：

It if it’s not working, you may wish to try using either rocm-opencl-runtime or opencl-amd. Mesa OpenCL implementations are unlikely to work.
如果仍然无效，您可能希望尝试使用 rocm-opencl-runtime 或 opencl-amd。Mesa OpenCL 实现不太可能工作。

If you still receive errors that the GPU is disabled, check if opencl-mesa is installed and remove them so they do not conflict.
如果您仍然收到 GPU 被禁用的错误，请检查是否安装了 opencl-mesa 并卸载它们以避免冲突。

If it still does not work, make sure that there is only one vendor listed in /etc/OpenCL/vendors; Folding@home will not work correctly if there are multiple vendors.
如果仍然无法工作，请确保 /etc/OpenCL/vendors 中只列出了一个供应商；如果存在多个供应商，Folding@home 将无法正常运行。
